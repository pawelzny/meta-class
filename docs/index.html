Installation

Install package with Composer:

I assume, you have composer in your working PATH


$ composer require pawelzny/meta-class

Basic Usage

Basic functionality is avzailable thanks to single Trait implemented in your class.

Thanks to Trait \Pawelzny\MetaClass\MetaClass any model gets access to public meta() method which is an interface for meta data. 


class User
{
    use \Pawelzny\MetaClass\MetaClass;

    // rest of User model
}

$user = new User;
$user->meta()->special_attribute = 'I am so special!';
$user->meta()->secretMethod = function () {
    return 'shh .. I am secret!';
};

Meta data initialization

\Pawelzny\MetaClass\MetaClass uses protected method metaInit() to initialize meta data on boot. It is invoke only once.

Can be override for custom initialization. It's a perfect place to initialize basic meta data on startup. 


class User
{
    use \Pawelzny\MetaClass\MetaClass;

    protected function metaInit()
    {
        $this->meta()->booted_args = ['start_at' => microtime(true)];
    }
}

$user = new User;
echo $user->meta()->booted_args['start_at'];

For booting complex meta data Components are more preferable.

Create Component

Components are preferable way to boot more complex logic. They are booted on startup like metaInit(), but uses composition instead of direct value assignment.

All components must implement Composable interface which exposes three public methods:

  • with(): Arguments supply
  • compose(): Perform computations
  • andReturn(): And return value


use \Pawelzny\MetaClass\Contracts\Composable;

class CustomComponent implements Composable
{
    protected $args = [];
    protected $output;

    /**
     * Args supplier
     */
    public function with(array $args = [])
    {
        $this->args = $args;

        return $this;
    }

    /**
     * Computations
     */
    public function compose()
    {
        // let keep it simple for example purposes
        $this->output = $this->args['value'] * 2;

        return $this;
    }

    /**
     * Return computed data
     */
    public function andReturn()
    {
        return $this->output;
    }
}

Composition of components

Compositions groups components and invoke one by one with given args list, then computed value is assign to meta data using component name as attribute name, or alias if defined.

Building Composition is as easy as extend \Pawelzny\MetaClass\MetaCompose class with defined components and arguments. 


use \Pawelzny\MetaClass\MetaCompose;
use \Pawelzny\MetaClass\Contracts\Composable;

class CustomComposition extends MetaCompose implements Composable
{
    /**
     * Custom component registry. Components are stacked
     * as fully qualified class name with namespace.
     * It may be plain string "\Nested\Namespace\CustomComponent",
     * but better option is to use magic constant.
     *
     * @var array $components
     */
    protected $components = [
        CustomComponent::class,
        'alias_name' =>  CustomComponent::class,
    ];

    /**
     * In this example, CustomComponent expected "value" argument.
     *
     * @var array $args
     */
    protected $args = ['value' => 23];
}

Example above represents implementation of custom meta composition class.

Compose invoke every component, passes args and gets it's computed value. Then the value is stored in meta attribute under namespace which is snake_case component's name.

In this example, first component will be registered under "custom_component" namespace, and second under "alias_name" namespace.

Using composition

Now it's time to override default Meta class and declare newly created CustomComposition. 


class User
{
    use \Pawelzny\MetaClass\MetaClass;

    /**
     * MetaClass trait checks if $meta_class attribute exists.
     * If yes, then creates Meta object from it instead of default MetaCompose class.
     * Because CustomComposition class has components and args, all will be invoke one by one
     * and result of components computation will be stored inside meta object.
     *
     * @var \Pawelzny\MetaClass\Contracts\Composable $meta_class
     */
    protected $meta_class = CustomComposition::class;

    // rest of User model
}

$user = new User;
assert($user->meta()->custom_component === 46); // 23 as input argument multiply by 2
